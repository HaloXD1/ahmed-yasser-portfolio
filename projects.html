01

Data engineering flagship

Retail Data Pipeline and KPI Dashboard

This turns messy retail data into trusted business KPIs. It shows I can move data from raw files to validation, storage, SQL models, and a dashboard.

How it works

  • Generates realistic retail CSV data.
  • Checks schemas, keys, duplicates, ranges, and relationships.
  • Loads clean records into a warehouse-style SQLite layer.
  • Builds SQL models for revenue, orders, products, returns, and regions.
  • Exports dashboard-ready files and a Streamlit KPI dashboard.

What it proves

Python, SQL, data quality, incremental loading, modeling, lineage, documentation, CI, and the ability to explain where bad data came from.

What to show first

Open the live dashboard, then show the quality diagnosis and lineage docs. That makes it feel like a real data engineering project, not just charts.

Interview line

"I built a small data pipeline that rejects bad rows before loading, models trusted KPIs in SQL, and documents the root cause of data quality issues."

02

Business systems simulation

OpsTwin Control Tower

This is a browser-based operations simulator. It lets a recruiter add disruptions, apply decisions, query the results, and generate a short incident postmortem.

How it works

  • Models suppliers, warehouses, products, customer regions, transport lanes, cash, and risk.
  • Runs a deterministic 26-week simulation.
  • Adds events like supplier delays, demand spikes, logistics failures, cash constraints, and data drift.
  • Applies decisions like safety stock, supplier diversification, expediting, rebalance, and demand shaping.
  • Uses charts, a network graph, in-browser SQL, and a generated postmortem to explain the outcome.

What it proves

React, TypeScript, product thinking, simulation logic, data visualization, testing, GitHub Pages deployment, and the ability to connect business tradeoffs with technical systems.

What to show first

Start on the command center, apply the recommended action, add a supplier delay on the scenario page, then open analytics or the postmortem.

Interview line

"I wanted one project that felt different from normal dashboards, so I built a static decision tool that simulates operational disruption and explains the tradeoffs."

React TypeScript Vite D3 DuckDB-Wasm Playwright

03

Analytics engineering

SaaS Analytics Engineering Pipeline

This simulates a SaaS company data stack. It turns product, subscription, and customer activity data into metrics like MRR, churn, adoption, and customer health.

How it works

  • Creates synthetic SaaS event, account, invoice, and subscription data.
  • Stores analytical data in DuckDB and Parquet.
  • Builds marts for revenue, churn, activation, adoption, and customer health.
  • Exports dashboard-ready metrics for a Streamlit demo.
  • Includes Docker, tests, CI, and documentation.

What it proves

Analytics engineering, dimensional thinking, metrics design, DuckDB, Parquet, customer analytics, and dashboard-ready model outputs.

What to show first

Show the dashboard metrics, then explain the model layer. It is strongest for BI, analytics, and data analyst internships.

Interview line

"This project models the SaaS metrics a product or revenue team would actually monitor: MRR, churn, adoption, and account health."

DuckDB Parquet SQL marts Streamlit Docker CI
Live demo Repository
04

Cloud security automation

Cloud Security Posture Auditor

This is an offline AWS-style security scanner. It checks cloud resource configurations against policy rules and produces risk scores, findings, and remediation steps.

How it works

  • Uses synthetic AWS-like resources, not real accounts.
  • Runs YAML policy checks for storage, IAM, network, encryption, and logging risks.
  • Scores findings by severity and affected account.
  • Generates remediation reports and dashboard views.
  • Includes secret scanning, tests, Docker, and CI.

What it proves

Security automation, cloud risk thinking, policy-as-code, reporting, and the ability to make a safe public project without exposing real cloud credentials.

What to show first

Show the dashboard and a remediation report. This is useful when speaking to cloud, security, or infrastructure teams.

Interview line

"I built an offline cloud posture auditor to show security automation without needing any real AWS account or secrets."

Python YAML policies Risk scoring Security reports Docker CI
