# Two-Week Internship Application Sprint

## Target Roles

- Data Engineering Intern
- BI Intern
- Data Analyst Intern
- Analytics Engineering Intern
- Cloud/Security Automation Intern
- Junior Python Automation Intern

## Companies To Target

Start with:

- Banks and fintech companies.
- Telecom companies.
- Retail and e-commerce companies.
- Consulting and audit firms with data teams.
- Software houses with data/BI roles.
- Cloud, cybersecurity, or managed-services companies.
- Multinational internship programs in Egypt, Gulf, Germany, and remote-friendly Europe.

Examples to search manually: Vodafone, Orange, Etisalat/e&, CIB, Banque Misr, QNB, Deloitte, PwC, EY, KPMG, IBM, Microsoft partners, SAP partners, Raya, Valeo, Instabug, Foodics, Noon, Talabat, and local analytics consultancies.

## Daily Plan

### Day 1

- Make public launch changes.
- Deploy Retail dashboard.
- Update tracker with 20 target companies.
- Read `standout_candidate_system.md`.

### Day 2

- Deploy SaaS dashboard.
- Deploy Cloud dashboard.
- Test all demo links.

### Day 3

- Update CV with live links.
- Update LinkedIn About and Featured.
- Pin GitHub repos.

### Day 4

- Apply to 5 Data Engineering or BI roles.
- Send 5 contact messages using `referral_outreach_system.md`.

### Day 5

- Apply to 5 more roles.
- Customize CV summary for two strongest companies.

### Day 6

- Practice retail and SaaS project pitches.
- Record yourself explaining each project once.

### Day 7

- Review tracker.
- Follow up on early applications where appropriate.
- Add 10 more companies.

### Day 8

- Apply to 5 roles.
- Message alumni/recruiters at 5 target companies.

### Day 9

- Apply to 5 roles.
- Practice 10 drills from `sql_interview_prep_pack.md`.

### Day 10

- Apply to 5 roles.
- Send follow-ups for Day 4 applications.

### Day 11

- Apply to cloud/security-adjacent internships.
- Use Cloud Security Automation CV version.

### Day 12

- Review interview answers.
- Prepare a 2-minute portfolio walkthrough.

### Day 13

- Apply to 5 more roles.
- Ask for referrals from trusted contacts.

### Day 14

- Audit tracker.
- Mark next follow-up dates.
- Improve weakest application materials based on responses.
- Choose DP-900 exam date only if budget and practice scores are ready.

## Follow-Up Schedule

- Day 0: Apply.
- Day 2: LinkedIn connection if no recruiter contact exists.
- Day 5-7: Short follow-up.
- Day 14: Final polite follow-up.

## Tracker Workflow

Use `docs/application_tracker.csv`.

Statuses:

- `Target`
- `Applied`
- `Followed up`
- `Interview`
- `Rejected`
- `Offer`
- `No response`

Update every application with:

- Date applied.
- Source.
- Contact person.
- Follow-up date.
- CV version used.
- Notes from the job description.
