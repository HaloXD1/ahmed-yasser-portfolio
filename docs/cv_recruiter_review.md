# CV Recruiter Review

## Verdict

The CV is strong for a junior Data Engineering / BI internship because it now has a clear technical portfolio story. It is project-heavy, but that is acceptable for a student with limited professional experience.

## Risk

The main risk is trying to show too many directions at once:

- Data Engineering
- BI/Data Analysis
- Cloud Security
- Python automation

The fix is not removing projects. The fix is choosing the right CV version for each role.

## Default CV Version

Use the Data Engineering CV as the default.

Project order:

1. Retail Data Pipeline and KPI Dashboard
2. SaaS Analytics Engineering Pipeline
3. Cloud Security Posture Auditor
4. Outlook to Google Calendar Scanner

## BI/Data Analyst Version

Emphasize:

- KPI dashboards
- business metrics
- Streamlit/Plotly/Tableau/Power BI
- data cleaning
- reporting

Put dashboard and metric language before pipeline internals.

## Cloud/Security Automation Version

Emphasize:

- Cloud Security Posture Auditor
- IAM, S3, network exposure, encryption, logging
- policy-as-code
- Python automation
- risk scoring and remediation

Keep Retail/SaaS as proof of Python and data skills.

## Public Version After Demos

After Streamlit demos are deployed:

- Add portfolio site URL in header or GitHub profile section.
- Add live demo links only if space allows.
- Keep the CV to one page.
- Do not add every GitHub link inside the CV if it makes it crowded.
- Prefer one GitHub profile link plus portfolio landing page.

## Recruiter Notes

The strongest claim is not “I know many tools.” The strongest claim is:

> I can build tested Python/SQL workflows that turn messy inputs into trusted outputs and dashboards.
