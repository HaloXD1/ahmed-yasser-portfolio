# Career Goals Hub

Last updated: 2026-05-13

## Operating Rules

- Main career lane: Data Engineering / Analytics Engineering.
- Secondary lane: Cloud Security Automation when it strengthens the data story.
- Market priority: Egypt and Gulf first; selected Europe and remote roles only when realistic.
- Weekly load: 10-15 hours.
- Application mode: prepare only. Ahmed submits manually.
- Portfolio proof order: live demos, then links and docs, then code quality.

## Goal 1: Internship Application Machine

Purpose: convert portfolio proof into interviews.

Outputs:

- Add or update 10-20 target roles when useful.
- Score roles with the weighted formula below.
- Produce top 5 apply-ready roles.
- Draft tailored CV angle, project bullets, and recruiter message.
- Set follow-up dates and update `application_tracker.csv`.

Ranking formula:

- 40% role fit: Data Engineering / Analytics Engineering / BI/Data.
- 20% practical market fit: Egypt, Gulf, or realistic remote.
- 15% deadline urgency.
- 15% tech match: Python, SQL, ETL, dashboards, cloud basics.
- 10% company signal: brand, team quality, learning value.

## Goal 2: Portfolio Proof Upgrade

Purpose: make recruiter clicks convert.

Outputs:

- Confirm or create next steps for live Streamlit demos.
- Update links only after they are real and tested.
- Improve README, CV, LinkedIn, and portfolio positioning.
- Run repo checks before claiming project stability.

Verification commands:

- Retail: `make test`, `make lint`, `make smoke`.
- SaaS: `make test`, `make lint`, `make smoke`.
- Cloud: `make test`, `make lint`, `make generate`, `make scan`, `make health`.

## Goal 3: Data Engineering Skill Sprint

Purpose: close the gap between portfolio and interview confidence.

Outputs:

- 2 skill tasks.
- 1 mini deliverable.
- 5 interview questions.
- 1 weak-skill note.
- Progress entry in `goal_runs/`.

Primary sequence:

SQL -> Python pipelines -> modeling -> dbt -> Airflow -> Docker/cloud -> interview defense.
