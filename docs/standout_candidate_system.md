# Standout Candidate System

Goal: look like a focused junior data engineer, not a student with random projects.

## Positioning

Default lane: Data Engineering / Analytics Engineering.

Backup lane: BI/Data Analyst roles with SQL and Python.

Secondary signal: Cloud Security Automation when the role values cloud, risk, IAM, or policy automation.

## Five Pillars

| Pillar | Output | Proof |
|---|---|---|
| Flagship project upgrade | Retail repo gets stronger demo, walkthrough, production plan | README, dashboard, docs, tests |
| SQL/interview prep | 40 SQL drills + answer patterns + project defense | `sql_interview_prep_pack.md` |
| Referral/outreach system | Apply + message + follow-up workflow | tracker, templates, target list |
| GitHub polish | Profile README + pinned repos + repo READMEs | `HaloXD1/README.md`, project READMEs |
| One light cert | DP-900 study path | `certification_plan_dp900.md` |

## Weekly Operating Rhythm

- Sunday: choose 10 roles and 10 contacts.
- Monday-Wednesday: apply to 2-3 roles/day.
- Tuesday-Thursday: send 2-3 referral messages/day.
- Friday: SQL practice + project pitch recording.
- Saturday: improve one portfolio artifact.

## Top Candidate Rule

Every application should include one sharp proof link:

- Data Engineering role: Retail pipeline or SaaS pipeline.
- BI/Data Analyst role: dashboard + KPI definitions.
- Cloud/Security role: Cloud Security Posture Auditor.
- Python automation role: Outlook Calendar Automation.

## 14-Day Outcome Target

- 25-35 quality applications.
- 20 referral or recruiter messages.
- 10 follow-ups.
- 40 SQL drills attempted.
- 3 project pitches recorded once.
- GitHub profile README updated.
