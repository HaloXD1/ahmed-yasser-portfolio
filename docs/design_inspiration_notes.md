# Design Inspiration Notes

## Sources Reviewed

- Portfolio Gallery: https://www.portfoliogallery.dev/
- Catalino Ong data portfolio: https://www.catalino.me/
- FounderJar data analytics portfolio examples: https://www.founderjar.com/inspiration/data-analytics-portfolio-examples/
- Creative Bloq portfolio examples: https://www.creativebloq.com/portfolios/examples-712368
- CreaDev Design case study: https://www.creadevdesign.com/projects/creadev-design

## Takeaways

- Generic project grids are common, so the new site should guide the visitor through proof rather than only list projects.
- Case-study framing is stronger than screenshot-only framing.
- A clean portfolio works best when the work is technical and functional.
- The site needs one memorable concept. For this portfolio, the concept is `Data Reliability Lab`.
- Recruiters need fast links: proof sheet, case study, GitHub, and role-specific evidence.

## Applied Direction

- Replaced the generic portfolio with a reliability-lab interface.
- Made the first screen about evidence: contracts, models, lineage, diagnosis, and CI.
- Added a role switchboard for Data Engineering, BI/Analytics, and Cloud Security.
- Made the retail project the flagship system, with SaaS and cloud security as supporting systems.
- Added a pipeline map to show how proof moves from raw data to dashboard.
